Configure


Once you have unpacked the download, you will need to add <GAFFER_INSTALL_PATH>/bin to your $PATH environment variable (substituting <GAFFER_INSTALL_PATH> with the path to the unpacked directory).

For example, you can achieve this in a bash shell with the following command:

$ export PATH=/home/jblogs/Downloads/gaffer-0.26.0.0/bin:${PATH}

That's it! You can launch Gaffer and start rendering with Appleseed:

$ gaffer

If you'd rather render with Arnold in Gaffer you will need to:

  • Ensure <ARNOLD_INSTALL_PATH>/bin is in your $PATH.
  • Add <ARNOLD_INSTALL_PATH>/bin to the $LD_LIBRARY_PATH environment variable.
  • Add <ARNOLD_INSTALL_PATH>/python to the $PYTHONPATH environment variable.

Similarly, to use 3delight with Gaffer you will also need to:

  • Ensure <3DELIGHT_INSTALL_PATH>/bin is in your $PATH.
  • Set the $DELIGHT environment variable to point to <3DELIGHT_INSTALL_PATH>.
  • Add <3DELIGHT_INSTALL_PATH>/shaders to the $DL_SHADERS_PATH environment variable.
  • Add <3DELIGHT_INSTALL_PATH>/displays to the $DL_DISPLAYS_PATH environment variable.
  • Add <3DELIGHT_INSTALL_PATH>/lib to the $LD_LIBRARY_PATH environment variable.

Tinker


A good first step would be working through the Quick Start Tutorial in the user documentation. It covers the basics of working with Gaffer to create and render a scene, taking you from launching the application through to viewing a complete image.
