1. Download

To install a pre-built release, simply download and unpack the latest bundle from the project site. The resulting directory will contain the complete application ready to run.

For linux, you will want the gaffer-VERSION-linux.tar.gz bundle. It's recommended that you choose the latest available version. Once you have this file downloaded, unpack it with the command (substituting in the correct version number):

$ tar -xvf gaffer-0.79.0-linux.tar.gz

2. Configure

Once you have unpacked the bundle, you will need to add the following location within the folder structure that gets created to your $PATH environment variable (substituting <GAFFER_INSTALL_PATH> with the path to the unpacked directory):

<GAFFER_INSTALL_PATH>/bin

For example, you can achieve this in a bash shell with the following command:

$ export PATH=/home/jblogs/Downloads/gaffer-0.79.0/bin:${PATH}

Then to use 3delight with Gaffer you will also need to:

  • Ensure <3DELIGHT_INSTALL_PATH>/bin is in your $PATH.
  • Set the $DELIGHT environment variable to point to <3DELIGHT_INSTALL_PATH>.
  • Add <3DELIGHT_INSTALL_PATH>/shaders to the $DL_SHADERS_PATH environment variable.
  • Add <3DELIGHT_INSTALL_PATH>/displays to the $DL_DISPLAYS_PATH environment variable.
  • Add <3DELIGHT_INSTALL_PATH>/lib to the $LD_LIBRARY_PATH environment variable.

Similarly, to use Arnold with Gaffer you will need to:

  • Ensure <ARNOLD_INSTALL_PATH>/bin is in your $PATH.
  • Add <ARNOLD_INSTALL_PATH>/bin to the $LD_LIBRARY_PATH environment variable.
  • Add <ARNOLD_INSTALL_PATH>/python to the $PYTHONPATH environment variable.

3. Tinker

A good first step would be working through the "QuickStart" tutorial in the user documentation. It covers the very basics of working with Gaffer to create and render a scene, taking you from launching the application through to viewing a complete image.
